# Unidade Lógico-Aritmética (ULA)

## Descrição do Problema

A **ULA (Unidade Lógico-Aritmética)** é a parte da CPU responsável por executar operações **aritméticas** e **lógicas**. Neste projeto, foi implementada uma ULA de **8 bits** capaz de realizar **adição, subtração e comparações**.

## Estrutura da ULA

A ULA implementada possui as seguintes **entradas e saídas**:

### **Entradas:**
- **A e B**: Operandos de 8 bits.
- **OP**: Define a operação (0 para soma, 1 para subtração). Comparações são sempre calculadas.

### **Saídas:**
- **R**: Resultado da operação aritmética (8 bits).
- **SignedOverflow**: Indica **overflow com sinal**.
- **UnsignedOverflow**: Indica **overflow sem sinal**.
- **Comparadores**:
  - **c0**: `1` se **A > B** (sem sinal).
  - **c1**: `1` se **A < B** (sem sinal).
  - **c2**: `1` se **A == B**.
  - **c3**: `1` se **A == 0**.
  - **c4**: `1` se **A > B** (complemento de dois).
  - **c5**: `1` se **A < B** (complemento de dois).

A implementação seguiu a abordagem **modular**, com os seguintes subcircuitos:

1. **Comparador**: Responsável por calcular os flags de comparação.
2. **Somador**: Reutilizado do EP1, realiza adição e subtração (via complemento de dois).
3. **ULA**: Integra os módulos anteriores e gera as saídas.

### **Detecção de Overflow**
- **Overflow com sinal**: Ocorre quando há mudança inesperada no bit de sinal.
- **Overflow sem sinal**: Baseia-se no carry out, mas requer ajustes para casos especiais.
