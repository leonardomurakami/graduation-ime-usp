## Descrição do Problema

Este exercício propõe a implementação de um algoritmo para **identificar e rotular as componentes biconexas** de um grafo não direcionado. O objetivo é processar a estrutura do grafo em **tempo O(n + m)** e classificar cada aresta de acordo com a componente biconexa à qual pertence.

Além disso, o programa deve ser capaz de identificar **pontos de articulação (cut vertices)** e **pontes (bridges)** quando solicitados.

## Formato de Entrada e Saída

O programa recebe como entrada:

1. Um número **d** que indica o nível de depuração (0 = componentes biconexas, 1 = cut vertices, 2 = bridges).
2. Dois inteiros **n** e **m**, representando o número de vértices e arestas, respectivamente.
3. **m** linhas contendo dois inteiros, representando uma aresta entre dois vértices.

A saída será:
- Para **d = 0**: Os rótulos das componentes biconexas de cada aresta.
- Para **d = 1**: Os vértices que são pontos de articulação.
- Para **d = 2**: As arestas que são pontes.

## Implementação

A implementação foi feita em **C++14**, utilizando a **Boost Graph Library (BGL)** apenas para armazenamento do grafo, conforme exigido pelo enunciado. O algoritmo de Tarjan foi utilizado para encontrar as **componentes biconexas** e determinar **pontos de articulação** e **pontes**.
